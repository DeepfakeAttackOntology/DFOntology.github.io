Abstract

The paper develops the first deepfake domain ontology that could assist common individuals in understanding the growing concerns of AI-manipulated digital media and researchers in domain knowledge integration and inference. The Deepfake Attack Ontology describes essential information related to a deepfake attack scenario, such as deepfake creator, victim, manipulated multimedia, used algorithms, training data, attack motivation, consequences, preventive measures, and more.

Deepfake Attack Ontology: Description back to ToC

The document explains the ontology for deepfake attacks. This is the first ontology in the deepfake domain and is developed using Web Ontology Language (OWL). The deepfake ontology provides a set of classes, properties, relations, and restrictions that can represent deepfake attacks in various contexts. The representation can help model essential information in a particular scenario like attack consequences and preventive measures. The first deepfake ontology can be enhanced by adding auxiliary entities and properties to model deepfake technology information for specialized applications.
